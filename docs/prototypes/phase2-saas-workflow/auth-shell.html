Change 1 / saas-auth-shell-permissions
登录后只信 `/api/auth/me`
这是整个 SaaS 前端的入口：当前是否登录、是谁、有哪些权限，都从这里拿。
验证入口：登录 / 刷新 / 菜单显隐
登录态判断 唯一业务入口
打开应用
GET /api/auth/me
进入主界面或登录页
`/api/auth/status` 只做兼容和启动探测，不再作为菜单和登录态主依据。
权限返回示例 Admin 可见
{ user: "13800138000", roles: ["管理员"], permission_flags: [ "admin:user:list", "admin:role:list" ], derived: { can_manage_users: true, can_manage_roles: true } }
你验证时看什么
场景预期说明
未登录打开应用跳到登录页不应靠 status 判断业务登录态
登录后刷新页面通过 /api/auth/me 恢复用户AuthContext 有 currentUser 和 permissionFlags
权限不足用户Admin 入口隐藏后端 403 仍是最终安全边界
