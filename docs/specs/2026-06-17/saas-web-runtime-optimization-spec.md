# SaaS Web Runtime Optimization Spec

## 背景

当前 `AionWeb` 来自 `AionUi` 迁移，Web 运行面仍带有桌面壳层残留、启动期副作用和样式节奏不统一的问题。当前版本功能可用，但存在请求偏多、启动链偏重、交互卡顿与视觉不对称。

## 目标

1. SaaS Web 启动链不再默认执行桌面专属副作用。
2. 首屏仅请求当前可见页面真正需要的数据，减少无效 eager fetch。
3. 侧栏与设置导航的节奏、对齐、选中态表现保持一致。
4. 优化不得破坏当前正常工作的登录、guid、conversation、settings 基础能力。

## 非目标

- 不重写路由、状态管理或组件体系
- 不做后端接口改造
- 不一次性重构整个 `src/aionui` 目录

## 约束

- 任何看起来像桌面能力的 API，都要先判断是否已被 SaaS backend bridge 复用
- 关键路径优先级高于“理论纯化”
- 以最小改动达成收益，不顺手做无关清理

## 验收标准

### 运行时纯化

- Web 启动不再默认挂载 deep-link、通知点击、tray、主进程日志、桌面 DevTools 等专属副作用
- 对仍需保留的 browser + backend bridge 能力不做误删

### 请求优化

- 启动链移除非首屏必要的 eager 请求或修复动作
- 同一类共享数据尽量复用缓存，不重复发起等价请求

### 样式修正

- 设置页侧栏分组标题、列表项与选中态视觉节奏对齐
- 对话侧栏分组、分割线、列表项间距避免明显不对称

### 性能

- 降低不必要的 mount 副作用与常驻监听
- 降低全局布局和侧栏在非目标页面上的无意义数据联动

## 验证

- 通过 `npm run typecheck`
- 通过 `npm run build`
- 对照代码路径记录已删、已延后、已保留的关键项

## 实际结果

- 已完成关键启动链纯化、cron 重复请求去重、Guid 可延后目录加载、设置/侧栏样式节奏对齐
- 已从 Web 入口移除 `webui` / `pet` / 通知点击等 SaaS 无关能力
- 构建验证已通过
- 全仓类型错误不作为本次优化完成门禁，按历史债务保留
